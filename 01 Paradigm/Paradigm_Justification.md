# Research Paradigm Justification

## 1. Paradigm Selection: Design Science

The research is based on the **Design Science methodology** because the main goal is not only to understand cybersecurity problems, but also to develop a practical solution for them. While approaches such as Positivism or Interpretivism help explain existing situations, Design Science focuses on creating and improving artifacts that can solve real-world problems. In cybersecurity, understanding how an attack works is important, however organizations also need effective tools that can help detect threats and respond to them as quickly as possible.

## 2. Research Goals and Proposed Solution

The main goal of this research is to help Security Operations Centers (SOCs) respond to cyber threats faster and more effectively. Many organizations still rely on fixed security processes that are often too slow when dealing with modern attacks.

To help solve this problem, the study develops two main components: an Autonomous Threat Hunting Agent, which uses Deep Learning to analyze security data and identify suspicious activities, and a Dynamic Playbook Optimization Layer, which helps choose the most appropriate response actions based on the threat situation.

As cyberattacks continue to evolve, traditional automated defenses are becoming less effective. This research does not only analyze the problem, but also proposes a practical solution that could improve incident response and reduce the Mean Time to Respond (MTTR) in SOC environments.

## 3. The Core Innovation: How Deep Learning and Playbooks Work Together

Unlike traditional approaches, this system does not see threat detection and incident response as two separate activities. The main idea is the continuous interaction between the Deep Learning(DL)agent and the response playbooks, allowing the system to adapt as the attack develops.

1. **Understanding the Attack Context:** The DL agent looks at security data over a period of time and tries to connect events that may seem unrelated at first. This helps build a clearer picture of what the attacker could be trying to achieve.
2. **Smarter Response Decisions:** Instead of asking security analysts to manually choose response actions during an incident, the agent helps guide and adjust the playbooks in real-time, so the response can better match the current threat situation.
3. **Learning from Experience:** The system continuously reviews the results of its actions and uses that information to improve future responses. Together with cybersecurity knowledge from experts, this helps the agent respond better when similar situations happen again and reduce unnecessary alerts.

## 4. Why Design Science Fits This Research

Design Science was selected as the research methodology because it aligns well with the objectives of this study. Since the research focuses on developing and evaluating a practical cybersecurity solution, this approach provides a suitable foundation for the work.

* **Positivist:** Mainly focuses on studying and testing existing situations rather than creating new solutions.

* **Interpretivist:** Helps understand human perspectives, but it is less suitable for developing a technical cybersecurity solution.

* **Critical Theory:** Usually focuses on social and organizational issues, which are not the main concern of this project.

* **Data-Intensive:** Works well for finding patterns in data, but this research also aims to build a system that can take action based on those findings.

* **Pragmatism:** Provides flexibility, but does not offer the same focus on designing and evaluating a new artifact.

## 5. Testing and Evaluation

An important part of this research is evaluating whether the proposed solution can actually improve incident response in a realistic cybersecurity environment. For this reason, the study includes a testing and evaluation phase focused on measuring its performance under different attack scenarios.

* **Prototype Development and Simulations:** A working prototype will be developed and tested in isolated environments designed to simulate real cybersecurity incidents and attack situations.

* **Performance Measurement:** The evaluation will focus on measuring key indicators, especially the Mean Time to Respond (MTTR), to understand if the solution can help security teams react more quickly.

* **Comparison with Traditional Approaches:** The results will be compared with more traditional security processes to see if the solution can improve response times while keeping false positive alerts at a reasonable level.

The overall goal is to better understand how this solution performs in practice and whether it can provide real benefits for Security Operations Centers.
